Ma Vision
Vous avez bâti une réputation solide (76/100 d'image). Mais dedans, trois choses manquent : (1) une direction écrite — le cap est dans votre tête, pas sur le papier ; (2) une délégation réelle — il se peut que vous portiez encore trop, laissant l'équipe en attente de clarté ; (3) une capitalisation du savoir — les recettes, les process, les décisions vivent dans les têtes, pas dans le système. Résultat : fatigue, turnover latent, qualité inégale. Vous avez la matière pour passer à 85/100 ; il manque l'architecture.
Palier 1 — redevenir cohérent avec son image
Frein : La stratégie 3 ans est dans la tête du dirigeant (3/5), pas écrite ; l'équipe ne connaît pas la direction (3/5). Résultat : chacun improvise, la qualité varie, les décisions traînent.
Écrire la stratégie 3 ans en 2 jours (atelier dirigeant + top 3 équipe) : vision, positionnement, 3 priorités annuelles, indicateurs clés. Afficher publiquement. Revoir les rôles et périmètres (qui décide quoi, qui fait quoi). Humain
→ Atelier stratégie (4h, facilitateur externe ou interne) → document 1 page + affichage → réunion d'équipe (30 min) pour présenter et recueillir les questions. Puis fiche de rôles par poste (1 page/personne).
Stratégie écrite et affichée ; 100% de l'équipe peut réciter les 3 priorités annuelles ; zéro ambiguïté sur qui décide quoi (mesure via entretien rapide).
Frein : Les façons de faire ne sont pas écrites (3/5) ; le savoir-faire vit dans les têtes (3/5). Chaque nouveau recruté réapprend ; chaque départ emporte du savoir. Ambiance de travail tendue (3/5) par manque de clarté.
Capitaliser les 5 à 7 process clés en fiches simples (1 page max/process) : accueil client, prise de commande, préparation cuisine, livraison, gestion des réclamations, fermeture. Impliquer l'équipe (elle sait, elle documente). Humain
→ Ateliers par process (30 min/process, équipe concernée) → rédaction de fiches (photos, étapes numérotées, pièges à éviter) → test avec un nouveau recruté ou un remplaçant → ajustement. Stockage centralisé (drive, wiki, ou papier plastifié en cuisine).
5 à 7 fiches process finalisées et testées ; délais tenus à 95%+ ; qualité constante (zéro réclamation liée à un oubli de process) ; nouveau recruté autonome en 2 semaines au lieu de 4.
Frein : Le dirigeant délègue peu (3/5) ; il porte trop. Il se peut que l'équipe attende un signal de confiance et que le climat en souffre (3/5). Recrutement dans l'urgence (3/5) = mauvais choix.
Identifier 2 à 3 tâches que vous pouvez déléguer immédiatement (ex : gestion des stocks, suivi des fournisseurs, planning des équipes). Confier à une personne clé avec formation et suivi hebdo (15 min). Anticiper les recrutements (6 mois avant). Humain
→ Audit rapide de votre agenda (où passez-vous 50% de votre temps ?) → identifier 3 tâches à déléguer → former la personne (1 semaine) → suivi hebdo (15 min) → ajustement. Créer un calendrier de recrutement annuel (qui part ? qui arrive ? quand former ?).
Vous libérez 5 à 8 heures/semaine ; la personne déléguée se sent confiante (entretien rapide) ; zéro urgence de recrutement ; ambiance de travail s'améliore (mesure via petit sondage équipe).
Palier 2 — dépasser grâce à l'IA
Frein : Les tâches répétitives (devis, factures, relances, comptes rendus clients) sont lentes et manuelles (3/5) ; les données sont éparpillées (3/5) ; les outils ne communiquent pas (3/5). Chacun double-saisit, perd du temps, fait des erreurs.
Automatiser les tâches répétitives via IA légère : (1) génération de devis/factures à partir d'un modèle + IA (ex : ChatGPT + Zapier, ou Make) ; (2) relances clients automatiques (SMS/email après commande, après 7 jours sans retour) ; (3) centraliser les données (CRM simple ou feuille Google partagée) ; (4) connecter les outils (caisse → facture → CRM, sans double saisie). IA
→ Audit des tâches répétitives (1 jour) → choisir 2 à 3 à automatiser en priorité (devis + relances) → implémenter un workflow IA simple (ChatGPT + Zapier, ou Make, ou un CRM léger type Brevo) → tester sur 1 semaine → former l'équipe (30 min) → déployer. Centraliser les données clients (nom, tel, adresse, historique commandes) dans un seul outil.
Temps de génération d'un devis : de 15 min à 2 min ; relances automatiques lancées (mesure du taux de réponse) ; zéro double saisie (données centralisées) ; 5 à 8 heures/semaine libérées pour l'équipe.
Frein : Les décisions s'appuient au feeling (3/5), pas sur des données. On ne sait pas précisément ce qui distingue La Romana (3/5) ; on ne surveille pas le marché (3/5). Chiffrage/devis sans méthode objective (3/5).
Mettre en place 3 à 4 indicateurs clés simples : (1) panier moyen par client (pour optimiser les offres) ; (2) taux de fidélité (% de clients qui reviennent) ; (3) coût de revient par plat (pour fixer les prix objectivement) ; (4) satisfaction client (note simple post-commande). Revoir votre positionnement : qu'est-ce qui vous rend unique ? (qualité, vitesse, ambiance, prix ?). Décider objectivement sur ces données, pas au feeling. IA
→ Définir les 3 à 4 KPI (1 jour) → les mesurer manuellement pendant 1 mois (pour établir une baseline) → automatiser la collecte (IA pour analyser les avis, CRM pour les données clients) → tableau de bord simple (feuille Google ou Metabase) → réunion mensuelle pour décider sur les données.
3 à 4 KPI mesurés et affichés ; décisions prises sur données (ex : prix ajusté, offre créée) ; panier moyen augmente de 5 à 10% ; taux de fidélité connu et amélioré.
